객체.메서드.속성

    메서드 종류
    getElementById() – id로 요소 찾기
    getElementsByClassName() – class로 요소들 찾기 (HTMLCollection 반환)
    querySelector() – CSS 선택자 하나를 가져오기
    querySelectorAll() – CSS 선택자에 해당하는 모든 요소 가져오기 (NodeList)
    createElement() – 새로운 HTML 요소 생성
    appendChild() – 자식 요소 추가
    remove() – 요소 삭제
    addEventListener() – 이벤트(클릭 등) 추가

    속성 종류
    innerText – 요소 안의 텍스트를 가져오거나 변경 (일반 HTML 요소)
    innerHTML – 요소 안의 HTML 코드를 가져오거나 변경 (일반 HTML 요소)
    value – 입력된 값을 가져오거나 변경 (input, textarea, select 등 폼 요소)
    style – 인라인 CSS 스타일 설정 (모든 HTML 요소)
    className – class 값을 가져오거나 변경 (모든 HTML 요소)
    id – id 값을 가져오거나 변경 (모든 HTML 요소)
    src – 이미지, 스크립트 등의 경로 지정 (img, script, iframe 등)
    href – 링크 이동 주소 설정 (a, link 등)



파일 모드
    "r"(읽기) - 파일을 읽기 전용으로 열며, 파일이 반드시 존재해야 한다
    "w"(쓰기) - 기존 내용을 모두 삭제하고 새로 작성, 파일이 없으면 생성된다, 덮어쓰기
    "a"(추가) - 기존 파일 내용 뒤에 데이터를 추가하며, 파일이 없으면 생성된다
    "x"(생성) - 새로운 파일을 생성하며, 이미 존재하면 오류가 발생한다

문자열 메서드
    rstrip() - 문자열 오른쪽(뒤쪽) 공백 제거    
    ex) "Hello\n".rstrip() → "Hello"

    lstrip() - 문자열 왼쪽(앞쪽) 공백 제거
    ex) " Hello".lstrip() → "Hello"

    split() - 구분자로 문자열 분할(리스트 반환)
    CSV 처리 시 split(',') 등 다른 문자열 메서드도 매우 유용
    ex) "A,B".split(',') → ["A", "B"]

    replace() - 특정 문자 치환
    ex) "2025-01".replace('-', '/') → "2025/01"

    lower() / upper() - 소문자/대문자 변환
    ex) "Hi".upper() → "HI"
        "Hi".lower() → "hi"

최초 Git 연결 요약
ex)
원격 저장소: Test (GitHub) 생성
로컬 폴더: test (PC)

    1. GitHub에서 저장소 생성
        -GitHub 로그인
        -New repository
        설정
            -Repository name: Test ← (대문자 T)
            -Public / Private 선택
            -README 생성 체크하지 않음 (중요)
        -Create repository     
    2️. cd test (로컬 test 폴더 진입)
    3️. git init (Git 초기화)
    4️. git add . (파일 추가)
    5. git commit -m "Initial commit" (첫 커밋 (필수))
    6. git branch -M main (브랜치 이름 통일)
    7. git remote add origin https://github.com/5EuiChan/Test.git (원격 저장소 연결)
    8️. git push -u origin main(최초 푸시@@@@@@@@@)

저장소 연결 후 코드 수정
[최초 작업시작]
    1. cd 작업할 파일 이름
    2. git clone https://github.com/이름/파일이름.git

[다른 PC에서 작업함]
    1. git pull   ← 항상 시작
    2. git status
    3. 코드 수정
    4. git add .
    5. git commit -m "설명"
    6. git push

방법 1️⃣ (이미 b 폴더가 있을 때 — 권장)
